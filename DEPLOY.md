# Deploy

How to run Havruta locally and how it deploys to GitHub Pages. The app is a static Progressive Web App with no backend. There are no server secrets and no environment variables, because the AI partner runs in the reader's own browser with the reader's own key.

## Run locally

You need Node 18 or newer.

```sh
npm install
npm run dev
```

Vite prints a local URL (usually http://localhost:5173). Open it in a browser. The app reads today's daf from Sefaria, so the dev machine needs internet access for the study screen to load text and the page image.

To check the production build:

```sh
npm run build
npm run preview
```

`npm run build` writes the static site to `dist/`. `npm run preview` serves that build so you can confirm it works before deploying. Run `npm run smoke` as well; it is the smoke test that also runs in CI on every push.

## Deploy on GitHub Pages

The app deploys to GitHub Pages from GitHub Actions. The live site is at https://oranburg.law/havruta/.

1. In the repository settings, under Pages, set the source to GitHub Actions.
2. Push to `main`. The workflow in `.github/workflows/deploy.yml` runs `npm ci`, `npm run build`, and `npm run smoke`, then deploys the `dist/` folder to Pages. The deploy is automatic on every push to `main`.
3. The Vite config sets `base: '/havruta/'`, so the built site lives under the `/havruta/` path. Keep that base in sync with the path the site is served from.
4. The custom domain `oranburg.law` is pointed at the GitHub Pages endpoint, and the path `/havruta/` matches the Vite base. If you serve the app from a plain `*.github.io/havruta/` URL instead, the same base still applies.

The app uses HashRouter, so routes live after the `#` in the URL and GitHub Pages needs no SPA rewrite rule for them to work. The PWA manifest and service worker are generated by vite-plugin-pwa into `dist/` at build time.

## Setting up the partner

There is nothing to configure on the server, because there is no server. A reader opens Settings in the app, chooses a provider, and pastes their own API key, which is stored only in that reader's browser. A reader with no key is pointed to the free Google Gemini path. The owner's key, if the owner uses one, lives only in the owner's browser, so a public visitor can never spend the owner's credits.

A future upgrade could add a server proxy (for example on Vercel) to offer the partner without a key for trusted accounts. That is not the current design, and it is not needed for the app to work.

## Regenerating the icons

The PWA icons are committed under `public/icons/`. To rebuild them from the brand colors:

```sh
python3 scripts/make_icons.py
```

This needs Pillow installed (`pip install Pillow`).
